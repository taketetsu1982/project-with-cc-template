## Why — 提供価値と目指す未来

### ビジョン

> Ouraのフィジカルデータとメンタル自己評価を掛け合わせ、心身の相互作用を可視化することで、日々の改善を実感できるダッシュボードを提供する。

Ouraは睡眠・活動・準備度というフィジカル面の優れた計測基盤を持つが、メンタルコンディションとの関連は見えない。本プロダクトはOuraデータを入力として受け取り、主観的メンタルスコアと統合した長期推移ダッシュボードを提供する。Ouraの「上位レイヤー」として機能する。

### 設計原則

| 原則 | 内容 | 根拠 |
|---|---|---|
| トレンド重視 | デフォルトビューは14日トレンド。日次スコアはドリルダウンのみ。数値よりカラーゾーン（緑/黄/赤帯）で範囲を見せる | Baron et al. 2017 (Orthosomnia)、Bergen Orthosomnia Scale 2025 の「硬直性」因子が日次数値への固執と相関 |
| 最小入力 | メンタル入力は4軸×7段階のスライダー。毎日30秒で完了できる設計 | EMAベストプラクティス: 4項目は疲労閾値（15項目・5分）の遥か内側 |
| 統合ビュー | フィジカルとメンタルを別々に見せない。ただしStrain（負荷）とRecovery（回復）は独立2軸で表示する | RESTQ: ストレスとリカバリーは独立次元。高ストレス＋高リカバリーがありうる |
| 個人ベースライン | 万人共通の基準ではなく、Z-score正規化で自分のベースラインからの偏差を見せる | 全主要ウェアラブル（Oura/WHOOP/Garmin）が採用。個人内変化が個人間差を上回る |
| 主観ファースト | メンタル入力を先に表示し、Ouraデータは後から重ねる。身体リテラシーを育て、デバイス依存を防ぐ | Orthosomnia緩和策: トラッカーの権威化を防止する |
| 観察的表現 | 「睡眠の規則性が高い週はストレスが低い傾向」のように、処方箋ではなく観察として相関を提示する | PG1ではAI提案を含まない。SDT（自己決定理論）の自律性を損なわない |

PG1（CPF）では個人利用のため、ビジネスモデルは未定義。まず自分自身の課題を解決できるかを検証する。

## Who — User（実際の利用者）

> PG1では Buyer = User = 自分自身。

### ターゲットUser

| ロール | 想定ユーザー | 主な利用シーン |
|--------|-------------|--------------|
| セルフトラッカー | Oura Ring所有者。フィジカルデータだけでは心身の全体像が見えないと感じている | 朝: メンタルスコア入力 → ダッシュボードで昨日の心身状態を確認。週末: 週次トレンドを振り返る |

### Userのペイン

1. **フィジカルとメンタルが分断されている**: Ouraで睡眠スコアが低い日にメンタルも不調だったのか、関連が見えない
2. **改善の実感が得られない**: 各スコアの日次変動はあるが、長期的に心身がどう改善しているかのトレンドが掴めない
3. **メンタル状態を多面的に記録できない**: Ouraの気分記録は単一ジェスチャー。気分・エネルギー・ストレス・集中を分けて追いたい
4. **スコアに囚われる**: 日次のスコアに一喜一憂してしまい、逆にストレスが増す（Orthosomnia）

## What — 機能定義

### 解決する課題

上記ペインに対し、以下の3つの軸で解決する:

1. **記録**: Ouraフィジカルデータの自動取得 + メンタル4軸の手動入力
2. **統合**: フィジカル×メンタルを一つのダッシュボードで可視化（Strain/Recovery分離）
3. **トレンド**: 日次ではなく14日/30日窓の変化パターンを強調

### スコープ

**プロダクト市場領域:** Oura Ringデータを活用した個人向け心身コンディション統合ダッシュボード

> PG1（CPF）スコープ: pg1.md を参照。AI改善提案・共有機能・目標設定は含まない。

### 機能概要

> 仮説: スコア項目・ダッシュボード構成は検証を通じて変わりうる

概念の定義・関係は `docs/reqs/conceptual-model.md` に記述する。

| 機能カテゴリ | 概要 |
|---|---|
| Ouraデータ連携 | Oura APIからSleep Score・Readiness Score・Activity Score・HRV（RMSSD）・安静時心拍数・睡眠ステージ比率を日次取得 |
| メンタルスコア入力 | 気分・エネルギー・ストレス・集中の4軸を1〜7のスライダーで日次入力 |
| 日次サマリー | その日のフィジカル（Oura）とメンタル（自己入力）を並べて表示。主観入力を先に、Ouraデータを後に重ねる |
| トレンドダッシュボード | 14日デフォルト窓（7日/30日切替可）で心身の推移を統合グラフで可視化。カラーゾーンで範囲を表示。相関パターンを観察的に提示 |

### メンタルスコア軸の設計根拠

> 仮説: 軸の選定は検証を通じて変わりうる

| 軸 | 測定対象 | 根拠 |
|---|---|---|
| 気分 (Mood) | 快/不快（Valence） | Russell's Circumplex Model のValence軸。最も検証された感情次元 |
| エネルギー (Energy) | 活力/疲労（Arousal） | Circumplex Model のArousal軸。Yale Mood Meterでも採用 |
| ストレス (Stress) | 主観的負荷感 | PSS-4のEMA適応版（Murray 2023）で日次使用が検証済み |
| 集中 (Focus) | 認知パフォーマンス | 日記研究で使用実績あり。睡眠不足の影響が大きい（Hedges' g = -0.776） |

**旧案（ストレス・モチベーション・集中・不安）からの変更理由**: ストレスと不安はCircumplex上でほぼ同一領域（高覚醒×ネガティブ）に位置し、簡易入力では弁別困難。モチベーションはエネルギー（覚醒度）で代替可能。気分とエネルギーはCircumplexの直交2軸として独立性が高い。

**スケール**: 7段階Likert（1-7）。5段階より信頼性・感度が有意に向上（Preston & Colman 2000）。VASより回答脱落率が低い（Haslbeck et al. 2025）。

### 正規化ロジック

フィジカル（ミリ秒単位のHRV等）とメンタル（1-7スケール）を統合比較するため、個人ベースラインZ-score正規化を採用する:

```
z = (今日の値 − 個人の14日移動平均) / 個人の標準偏差
```

これにより全メトリクスが「個人の平均からの標準偏差数」という同一単位になる。

### 時間窓

| 窓 | 用途 | デフォルト |
|---|---|---|
| 7日 | 短期フィードバック、平日/休日パターン | - |
| 14日 | トレンド検出。Oura Resilienceと同窓。信号のクリアさとトレンド可視性のバランス | **デフォルト** |
| 30日 | 累積ストレス評価、習慣形成 | - |

## 用語集

| 用語 | 定義 |
|---|---|
| メンタルスコア | 気分・エネルギー・ストレス・集中の4軸で自己評価する主観的指標（1〜7） |
| フィジカルスコア | Oura APIから取得するSleep Score・Readiness Score・Activity Score等の総称 |
| 統合ダッシュボード | フィジカルスコアとメンタルスコアの推移を一つのビューで可視化する画面 |
| Orthosomnia | スコアに囚われて逆にストレスが増す現象（Baron et al. 2017）。本プロダクトではトレンド重視UIで回避を目指す |
| Z-score正規化 | 各メトリクスを個人ベースラインからの標準偏差数に変換し、異なる単位のデータを統合比較可能にする手法 |
| Strain/Recovery | RESTQ由来の概念。負荷と回復は独立した2軸であり、同時に高い状態がありうる |
| Circumplex Model | Russell (1980) の感情円環モデル。Valence（快/不快）×Arousal（高/低覚醒）の2軸で感情を配置する |
