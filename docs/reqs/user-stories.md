## US-01: Oura APIを接続する

**優先度:** Must
**エピック:** Ouraデータ連携

### ストーリー

**セルフトラッカーとして**、Oura APIとOAuth2で接続したい。**なぜなら**、フィジカルデータ（Sleep Score・Readiness Score・Activity Score・HRV・安静時心拍数）を自動取得してダッシュボードに表示するためだ。

### 受け入れ条件

- [ ] Oura接続設定画面でOAuth2認証フローを開始できる
- [ ] 認証成功後、アクセストークンとリフレッシュトークンが保存される
- [ ] 接続状態が「接続済み」と表示される
- [ ] 接続成功後、Ouraからフィジカルデータの日次取得が開始される

### シナリオ

```gherkin
Given セルフトラッカーがOura接続設定画面を開いている
  And Oura APIが未接続の状態である
When 「Ouraを接続」ボタンをタップする
  And Oura OAuth2認証画面で認可する
Then 接続状態が「接続済み」に変わる
  And フィジカルデータの取得が開始される
```

### エッジケース

- OAuth2認証をキャンセルした場合、接続状態は変わらずエラーメッセージを表示する
- トークンの有効期限切れ時にリフレッシュトークンで自動更新する
- Oura APIが一時的に利用不可の場合、リトライまたはエラー状態を表示する

## US-02: Oura接続を解除する

**優先度:** Must
**エピック:** Ouraデータ連携

### ストーリー

**セルフトラッカーとして**、Oura APIとの接続を解除したい。**なぜなら**、必要に応じてアカウント連携を停止・再接続できるようにするためだ。

### 受け入れ条件

- [ ] Oura接続設定画面で「切断」操作ができる
- [ ] 切断後、接続状態が「未接続」に変わる
- [ ] 保存されたトークン情報が削除される
- [ ] 既に取得済みのフィジカルデータは保持される

### シナリオ

```gherkin
Given セルフトラッカーがOura接続設定画面を開いている
  And Oura APIが接続済みの状態である
When 「切断」ボタンをタップする
  And 確認ダイアログで「はい」を選択する
Then 接続状態が「未接続」に変わる
  And トークン情報が削除される
```

### エッジケース

- 切断後に再接続する場合、新たにOAuth2認証フローが必要
- 切断操作中にネットワークエラーが発生した場合のエラーハンドリング

## US-03: メンタルスコアを入力する

**優先度:** Must
**エピック:** メンタルスコア入力

### ストーリー

**セルフトラッカーとして**、気分・エネルギー・ストレス・集中の4軸を1〜7のスライダーで毎日入力したい。**なぜなら**、主観的なメンタルコンディションを記録してフィジカルデータと統合的に振り返るためだ。

### 受け入れ条件

- [ ] メンタルスコア入力画面で4軸（気分・エネルギー・ストレス・集中）のスライダーが表示される
- [ ] 各スライダーは1〜7の整数値を選択できる
- [ ] 当日分のメンタルエントリを新規作成できる
- [ ] 保存完了後、日次サマリー画面に遷移する
- [ ] 入力は30秒以内で完了できるシンプルなUIである

### シナリオ

```gherkin
Given セルフトラッカーがメンタルスコア入力画面を開いている
  And 当日のメンタルエントリがまだ存在しない
When 気分を5、エネルギーを4、ストレスを3、集中を6に設定する
  And 「保存」ボタンをタップする
Then メンタルエントリが当日の日次レコードに保存される
  And 日次サマリー画面に遷移する
```

### エッジケース

- 未来の日付に対してメンタルスコアを入力しようとした場合、入力を拒否する
- 当日分が既に入力済みの場合、編集モードで既存の値を表示する
- 4軸すべてに値を設定せずに保存しようとした場合の挙動（全軸必須）
- 1未満または7超の値は入力できない

## US-04: 過去のメンタルスコアを修正する

**優先度:** Must
**エピック:** メンタルスコア入力

### ストーリー

**セルフトラッカーとして**、過去に入力したメンタルスコアを修正したい。**なぜなら**、入力ミスや振り返りによる再評価を反映できるようにするためだ。

### 受け入れ条件

- [ ] 日次サマリー画面からメンタル編集ボタンで入力画面に遷移できる
- [ ] 過去のメンタルエントリの既存値がスライダーにプリセットされる
- [ ] 値を変更して保存できる
- [ ] 保存後、日次サマリー画面に戻る

### シナリオ

```gherkin
Given セルフトラッカーが3日前の日次サマリー画面を表示している
  And メンタルエントリが入力済みである
When 「メンタル編集」ボタンをタップする
  And ストレスの値を4から2に変更する
  And 「保存」ボタンをタップする
Then メンタルエントリが更新される
  And 日次サマリー画面に戻り更新後の値が表示される
```

### エッジケース

- 過去のメンタルエントリが存在しない日の場合、新規作成として扱う
- 変更せずに保存した場合、更新日時は変わらない

## US-05: 日次サマリーを確認する

**優先度:** Must
**エピック:** 日次サマリー

### ストーリー

**セルフトラッカーとして**、特定の日のメンタルスコアとフィジカルスコアを一つの画面で確認したい。**なぜなら**、その日の心身の状態を統合的に把握するためだ。

### 受け入れ条件

- [ ] 日次サマリー画面で特定日のメンタルエントリとフィジカルスコアが表示される
- [ ] メンタル（主観）が先に、フィジカル（Oura）が後に表示される（主観ファースト原則）
- [ ] 各値がZ-score正規化で個人ベースラインからの偏差として表示される
- [ ] メンタル未入力の場合、入力を促すCTAが表示される
- [ ] 「戻る」操作でトレンドダッシュボードに戻れる

### シナリオ

```gherkin
Given セルフトラッカーがトレンドダッシュボードを表示している
When 特定の日付をタップする
Then 日次サマリー画面に遷移する
  And メンタルスコア（気分・エネルギー・ストレス・集中）が上部に表示される
  And フィジカルスコア（Sleep Score・Readiness Score・Activity Score・HRV・安静時心拍数）が下部に表示される
```

```gherkin
Given セルフトラッカーが日次サマリー画面を表示している
  And 当日のメンタルエントリが未入力である
When 画面を確認する
Then メンタルスコア入力を促すCTAが表示される
```

### エッジケース

- Oura接続がない場合、フィジカルスコア欄に「Oura未接続」メッセージを表示する
- Ouraデータがまだ取得されていない日の場合、フィジカルスコア欄に「データなし」を表示する
- Z-score計算に十分なデータ（14日分）が蓄積されていない場合、生の値のみ表示する

## US-06: トレンドダッシュボードで心身の推移を確認する

**優先度:** Must
**エピック:** トレンドダッシュボード

### ストーリー

**セルフトラッカーとして**、フィジカルとメンタルの統合トレンドをダッシュボードで確認したい。**なぜなら**、日次の数値に一喜一憂せず長期的な心身の改善傾向を把握するためだ。

### 受け入れ条件

- [ ] デフォルトで14日間のトレンドが統合グラフで表示される
- [ ] カラーゾーン（緑/黄/赤帯）でZ-score範囲が視覚的に表示される
- [ ] フィジカルとメンタルが同一ビューで統合表示される
- [ ] Strain（負荷）とRecovery（回復）が独立した2軸で表示される
- [ ] 相関パターンが観察的表現（処方箋ではなく観察）で提示される
- [ ] 数値よりもトレンド（変化の方向性）が強調される
- [ ] 今日のメンタル未入力時、入力CTAが目立つ位置に表示される

### シナリオ

```gherkin
Given セルフトラッカーがトレンドダッシュボードを開いている
  And 14日分以上のデータが蓄積されている
When ダッシュボードを確認する
Then 直近14日間のフィジカル×メンタルの統合トレンドグラフが表示される
  And カラーゾーンでZ-score範囲が示される
  And 「睡眠の規則性が高い週はストレスが低い傾向」のような観察的な相関パターンが表示される
```

```gherkin
Given セルフトラッカーがトレンドダッシュボードを開いている
  And 今日のメンタルエントリが未入力である
When ダッシュボードを確認する
Then 「今日のメンタルを記録する」CTAが目立つ位置に表示される
```

### エッジケース

- データが14日分未満の場合、蓄積済みデータのみでグラフを表示する（空状態メッセージ付き）
- Oura未接続の場合、メンタルスコアのみでトレンドを表示し、接続を促すリンクを表示する
- 全期間メンタル未入力の場合、フィジカルデータのみ表示し入力を促す
- 標準偏差が0の軸（全日同一値）ではZ-scoreを計算できないため、該当軸は「変動なし」と表示する

## US-07: トレンドの時間窓を切り替える

**優先度:** Must
**エピック:** トレンドダッシュボード

### ストーリー

**セルフトラッカーとして**、トレンドの表示期間を7日・14日・30日で切り替えたい。**なぜなら**、短期の変化パターンと長期の改善傾向を使い分けて確認するためだ。

### 受け入れ条件

- [ ] 7日・14日・30日の時間窓切替UIが表示される
- [ ] デフォルトは14日が選択されている
- [ ] 切り替え時にグラフがリロードされ選択した期間のデータが表示される
- [ ] 選択中の時間窓が視覚的にハイライトされる

### シナリオ

```gherkin
Given セルフトラッカーがトレンドダッシュボードを14日表示で確認している
When 時間窓を「30日」に切り替える
Then グラフが直近30日間のトレンドに更新される
  And 「30日」が選択状態として表示される
```

### エッジケース

- 選択した期間よりデータが少ない場合、存在するデータのみ表示する
- 時間窓を切り替えてもZ-scoreの計算基準（14日移動平均）は変わらない

---

## US-08: ダッシュボードから日次サマリーにドリルダウンする

**優先度:** Must
**エピック:** トレンドダッシュボード

### ストーリー

**セルフトラッカーとして**、トレンドグラフ上の特定日をタップして日次サマリーに遷移したい。**なぜなら**、気になる日の詳細を素早く確認するためだ。

### 受け入れ条件

- [ ] トレンドグラフ上の日付ポイントをタップすると日次サマリーに遷移する
- [ ] 遷移先ではタップした日付の日次サマリーが表示される
- [ ] 日次サマリーから「戻る」でトレンドダッシュボードに戻れる

### シナリオ

```gherkin
Given セルフトラッカーがトレンドダッシュボードを表示している
When グラフ上の3日前のデータポイントをタップする
Then 3日前の日次サマリー画面に遷移する
  And その日のメンタルスコアとフィジカルスコアが表示される
```

### エッジケース

- データが存在しない日をタップした場合、空の日次サマリーを表示する（メンタル入力CTAとOuraデータなしメッセージ）
