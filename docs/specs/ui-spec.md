## デザイントークン

### カラー

| 用途 | トークン | 値 | 説明 |
|-----|---------|---|------|
| Z-score 良好帯 | `--zone-good` | `#4CAF50` (green) | Z ≤ −0.5（ストレス）/ Z ≥ 0.5（その他）|
| Z-score 中立帯 | `--zone-neutral` | `#FFC107` (amber) | −0.5 < Z < 0.5 |
| Z-score 注意帯 | `--zone-concern` | `#F44336` (red) | Z ≥ 0.5（ストレス）/ Z ≤ −0.5（その他）|
| Strain 軸 | `--strain` | `#FF7043` (deep orange) | Strain合成スコアの線・ラベル |
| Recovery 軸 | `--recovery` | `#42A5F5` (blue) | Recovery合成スコアの線・ラベル |
| メンタル系 | `--mental` | `#AB47BC` (purple) | メンタル4軸の共通アクセント |
| フィジカル系 | `--physical` | `#26A69A` (teal) | フィジカル系データの共通アクセント |
| CTA | `--cta` | Material Primary | メンタル入力CTAボタン |

**カラーゾーンの方向性:** ストレスのみ「高い=注意」。他の軸は「高い=良好」。Z-scoreの符号でゾーンを判定する。

### タイポグラフィ

- HTML lang: `ja`
- 日付フォーマット: `M/D（ddd）`（例: `3/19（水）`）
- 日付範囲: `M/D – M/D`（例: `3/6 – 3/19`）
- 数値: 小数第1位まで（Z-score表示時）

## 状態と UI の振る舞い

### ローディング

画面全体のデータ取得中は Material の `md-linear-progress`（indeterminate）をAppBar直下に表示。スケルトンは使わない（PG1ではシンプルさ優先）。

### エラー表示

| パターン | スタイル | 使用箇所 |
|---------|---------|---------|
| API通信エラー | インラインバナー（`md-filled-tonal-button` で再試行） | 全画面共通 |
| バリデーションエラー | フィールド下の `md-outlined-text-field` supporting text（赤） | メンタルスコア入力 |
| Oura APIエラー | インラインバナー + エラー詳細 | Oura接続設定 |

| 画面 | 条件 | メッセージ |
|------|------|----------|

## 画面詳細

### トレンドダッシュボード (`/`)

**API:** `GET /api/v1/trends?window={7|14|30}`

**レイアウト（上→下）:**

1. **メンタル入力CTA**（`today_mental_submitted === false` 時のみ）
   - `md-filled-button`「今日のメンタルを記録する」→ `/mental-entry`
   - 画面上部、グラフより上に固定表示

2. **時間窓セレクター**
   - `md-outlined-segmented-button-set` で 7日 / **14日** / 30日
   - デフォルト: 14日が選択状態

3. **Strain / Recovery グラフ**
   - 折れ線グラフ2本（Strain: `--strain` / Recovery: `--recovery`）
   - Y軸: Z-score（−2 〜 +2 目安）
   - X軸: 日付（`M/D` ラベル）
   - カラーゾーン: 背景に3帯（緑/黄/赤）を水平に描画
   - データポイントをタップ → `/daily/:date` に遷移

4. **相関パターン**（`correlations` 配列が非空の場合）
   - `md-list` で観察テキストをリスト表示
   - 例: 「睡眠スコアが高い日はストレスが低い傾向があります」

**表示ルール:**
- 数値ラベルはグラフ上に表示しない（トレンド重視・Orthosomnia回避）
- データポイントのツールチップ: タップ時のみ日付と大まかなゾーン（良好/中立/注意）を表示
- Z-score未算出期間（14日未満）: 生値の折れ線グラフに切り替え、カラーゾーンは非表示

---

### 日次サマリー (`/daily/:date`)

**API:** `GET /api/v1/daily-records/{record_date}`

**レイアウト（上→下）:**

AppBar: 日付タイトル（`M/D（ddd）`）+ 戻る矢印

1. **メンタルセクション**（主観ファースト — 必ず上部）
   - 未入力時: CTA「メンタルスコアを入力する」→ `/mental-entry/:date`
   - 入力済み: 4軸を縦並びカード表示

   | 表示 | 内容 |
   |------|------|
   | 軸名 | 気分 / エネルギー / ストレス / 集中 |
   | 生値 | 1〜7 のドットインジケータ（現在値をハイライト） |
   | Z-score | `+0.5` のようなバッジ。ゾーンカラーで背景色 |
   | Z-score未算出 | バッジ非表示（生値のみ） |

   - 右上に「編集」アイコン → `/mental-entry/:date`

2. **フィジカルセクション**（メンタルの下部）
   - 未接続: 「Oura未接続」メッセージ + 設定リンク
   - データなし: 「データなし」メッセージ
   - データあり: 各メトリクスをリスト表示

   | メトリクス | 表示形式 |
   |-----------|---------|
   | Sleep Score | 整数 + Z-scoreバッジ |
   | Readiness Score | 整数 + Z-scoreバッジ |
   | Activity Score | 整数 + Z-scoreバッジ |
   | HRV (RMSSD) | 小数1桁 ms + Z-scoreバッジ |
   | 安静時心拍数 | 小数1桁 bpm + Z-scoreバッジ |
   | 睡眠ステージ | 横棒グラフ（REM/Light/Deep/Awake の比率） |

**Z-scoreバッジ:**
- ゾーンカラー背景 + 白文字
- 標準偏差0の軸: 「変動なし」テキスト（バッジなし）
- 14日未満: バッジ非表示

---

### メンタルスコア入力 (`/mental-entry`, `/mental-entry/:date`)

**API:** `POST /api/v1/mental-entries`（新規）/ `PATCH /api/v1/mental-entries/{record_date}`（編集）

**レイアウト（上→下）:**

AppBar: 「メンタルスコア」+ 戻る矢印

1. **日付表示**（読み取り専用。当日 or パスパラメータの日付）

2. **4軸スライダー**（縦並び）

   各軸:
   ```
   気分 ───────────●─── 5
   1                    7
   ```
   - `md-slider`（discrete、step=1、min=1、max=7）
   - ラベル: 軸名（左）+ 現在値（右、大きめフォント）
   - 初期値: 新規=4（中央）/ 編集=既存値
   - 軸の順序: 気分 → エネルギー → ストレス → 集中

3. **保存ボタン**
   - `md-filled-button`「保存」
   - 保存成功 → `/daily/:date` に遷移

**バリデーション:**
- 4軸すべて必須（スライダーUIのため未入力は発生しない。初期値4で開始）
- API 409（当日分既存）→ 編集モードにリダイレクト

---

### Oura接続設定 (`/settings/oura`)

**API:** `GET /api/v1/oura/connection`, `POST /api/v1/oura/authorize`, `DELETE /api/v1/oura/connection`

**レイアウト（上→下）:**

AppBar: 「Oura接続設定」+ 戻る矢印

1. **接続状態カード**
   - 接続済み: 「接続済み」チップ（緑）+ トークン有効期限表示
   - 未接続: 「未接続」チップ（グレー）

2. **アクションボタン**
   - 未接続時: `md-filled-button`「Ouraを接続」→ OAuth2認証フロー開始
   - 接続済み時: `md-outlined-button`「切断」→ 確認ダイアログ後に `DELETE`

3. **確認ダイアログ**（切断時）
   - `md-dialog`「Oura接続を切断しますか？」
   - 説明: 「既に取得済みのデータは保持されます」
   - アクション: 「キャンセル」/「切断」

## インタラクション定義

### グラフのデータポイントタップ

1. タップしたポイントの日付を特定
2. ツールチップを0.3秒表示（日付 + ゾーン）
3. タップ後に `/daily/:date` に遷移

### 時間窓の切り替え

1. セグメントボタンの選択を変更
2. `GET /api/v1/trends?window={new}` を呼び出し
3. ローディングインジケータ表示
4. グラフを新データで再描画（アニメーション 300ms）

### メンタルスコア入力の保存

1. 「保存」ボタンタップ
2. ボタンを disabled + ローディング状態に
3. API呼び出し（POST or PATCH）
4. 成功: `/daily/:date` に遷移
5. 失敗: ボタン復帰 + エラーバナー表示

## コンポーネント規約

### 状態管理

- グローバル状態: Oura接続状態（`oura_connected`）、認証トークン
- 画面ローカル状態: フォーム入力値、選択中の時間窓、ローディング/エラー
- データキャッシュ: トレンドデータは時間窓ごとにキャッシュ。メンタル入力保存後にキャッシュ無効化

### APIクライアント

- ベースURL: `/api/v1`
- 認証: `Authorization: Bearer <access_token>` ヘッダーを自動付与
- トークンリフレッシュ: 401応答時に `POST /api/v1/auth/refresh` を自動実行し、リトライ
- エラーハンドリング: APIエラーレスポンス（`{ "detail": "..." }`）をパースし、UIのエラーバナーに表示
- 日付パラメータ: `YYYY-MM-DD` 形式で送信
